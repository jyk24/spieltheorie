Wo steckt der Fehler?

Das Tausch-Argument behandelt {{ amount }}€ als beliebigen Betrag X und sagt: Der andere Umschlag enthält X/2 oder 2X. Aber:

  • → Wenn dein Umschlag der kleinere ist ({{ amount }}€), enthält der andere {{ amount * 2 }}€. Tausch: +{{ amount }}€
  • → Wenn dein Umschlag der größere ist ({{ amount }}€), enthält der andere {{ (amount / 2) | int }}€. Tausch: -{{ (amount / 2) | int }}€
  • → Diese zwei Szenarien sind nicht symmetrisch: Du gewinnst {{ amount }} aber verlierst nur {{ (amount / 2) | int }}
Echter EV: 0.5 × (+{{ amount }}) + 0.5 × (-{{ (amount / 2) | int }}) = +{{ ((amount * 0.5) - ((amount / 2) * 0.5)) | round(1) }}€

Das Problem: Ohne Verteilung der möglichen Beträge zu kennen, kannst du den Erwartungswert nicht korrekt berechnen. Das Argument enthält eine verdeckte Annahme über die Vorverteilung.

Die tiefere Erklärung:

Sei X der kleinere Betrag (den weder du noch der andere Spieler kennt). Die Umschläge enthalten X und 2X. Du siehst entweder X oder 2X:

  • → Wenn du X siehst: Tausch gibt +X (Gewinn)
  • → Wenn du 2X siehst: Tausch gibt −X (Verlust)
  • → Symmetrisch: EV(Tausch) = EV(Behalten) = 1.5X

Der Fehler im ursprünglichen Argument: Es behandelt den gesehenen Betrag als fixen Ausgangspunkt, statt als zufällige Variable. Damit doppelt es die Unsicherheit auf falsche Weise.

Verhandlungsrelevanz:

Das Zwei-Umschlag-Problem zeigt: Wenn eine Entscheidung "immer vorteilhaft" erscheint, ist fast sicher ein Denkfehler im Spiel. In Verhandlungen taucht das auf, wenn jemand sagt: "Egal was du tust, dieses Angebot ist besser."

Regel: Beachte, welche Referenzpunkte das Argument nutzt. Argumente, die aus beliebigen Ausgangspunkten "immer" zu einer Schlussfolgerung führen, enthalten fast immer einen verdeckten Fehler.
